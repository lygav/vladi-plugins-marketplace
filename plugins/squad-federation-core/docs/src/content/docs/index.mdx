## Architecture

Squad Federation provides a **3-layer architecture**:

- **SDK Layer** - TypeScript interfaces and schemas
- **Lib Modules** - Onboarding, communication, monitoring implementations
- **Scripts** - CLI entry points for federation management

## Quick Example

```bash
# Create federation
npx tsx scripts/init-federation.ts --communication file-signal

# Onboard a coding team
npx tsx scripts/onboard.ts \
  --archetype coding \
  --domain backend-api \
  --placement worktree \
  --mission "Build REST API with PostgreSQL"

# Launch team
npx tsx scripts/launch.ts --team backend-api

# Monitor progress
npx tsx scripts/monitor.ts --watch
```

## Use Cases

**Multi-Module Projects** - Separate teams for frontend, backend, infrastructure, each with dedicated worktrees

**Documentation Pipelines** - Deliverable teams generate docs while coding teams implement features

**Code Review Workflows** - Consultant teams analyze code and provide recommendations

**Large Migrations** - Coordinate multiple teams refactoring different modules in parallel

## What's New in v0.5.0

- **Placement/Communication Separation** - Teams can now mix worktree and directory placements within one federation
- **Teams Channel Transport** - Real-time coordination via Microsoft Teams with hashtag protocol
- **Learning Log Format** - Standardized JSONL for capturing and sweeping team learnings
- **File Locking** - Thread-safe team registry updates prevent concurrent modification
