## Key Features

<CardGrid>
  <Card title="Specialized Teams" icon="puzzle">
    Create teams with specific archetypes (coding, deliverable, consultant). Each team has its own lifecycle states and skills.
  </Card>
  <Card title="Signal-Based Communication" icon="comment">
    Teams coordinate via file-based signals — fast, offline, debuggable. Optionally get meta-squad summaries in a Microsoft Teams channel.
  </Card>
  <Card title="Knowledge Lifecycle" icon="open-book">
    Capture learnings as teams work, sweep patterns, graduate reusable skills, and sync across domains automatically.
  </Card>
  <Card title="Placement Options" icon="setting">
    Deploy teams in worktrees, subdirectories, or custom locations. Mix placement types within one federation.
  </Card>
</CardGrid>

## Architecture

Squad Federation provides a **3-layer architecture**:

- **SDK Layer** - TypeScript interfaces and schemas
- **Lib Modules** - Onboarding, communication, monitoring implementations
- **Scripts** - CLI entry points for federation management

## Get Started

Talk to Copilot using the federation skills. Create teams conversationally, launch them, and monitor their progress—no manual commands needed.

## Use Cases

**Multi-Module Projects** - Separate teams for frontend, backend, infrastructure, each with dedicated worktrees

**Documentation Pipelines** - Deliverable teams generate docs while coding teams implement features

**Code Review Workflows** - Consultant teams analyze code and provide recommendations

**Large Migrations** - Coordinate multiple teams refactoring different modules in parallel
